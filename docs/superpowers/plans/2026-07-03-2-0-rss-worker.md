# 2.0 RSS and Worker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mock RSS pages with real feed source CRUD, feed entry storage, read-state APIs, and an idempotent worker fetch path.

**Architecture:** Web API routes own auth and request validation. `packages/db` repositories own user-scoped feed/feed-entry queries. `apps/worker` owns worker execution and uses queue payloads from `packages/queue`. The first fetch implementation parses RSS/Atom from XML and upserts by `feedId + url`.

**Tech Stack:** Next.js API Routes, Prisma, BullMQ, Node fetch, `fast-xml-parser`, Vitest.

---

## File Structure

- Create `apps/web/src/app/api/feeds/route.ts`: list/create feed sources.
- Create `apps/web/src/app/api/feeds/[id]/route.ts`: read/update/delete feed source.
- Create `apps/web/src/app/api/feeds/[id]/entries/route.ts`: list entries for a feed.
- Create `apps/web/src/app/api/feed-entries/[id]/route.ts`: read entry and mark read/unread.
- Modify `apps/web/src/app/(app)/feeds/page.tsx`: real feed source list.
- Modify `apps/web/src/app/(app)/feeds/[id]/page.tsx`: real entry list.
- Create `apps/web/src/app/(app)/feed-entries/[id]/page.tsx`: feed entry reader route.
- Modify `packages/db/src/repositories/feeds.ts`: add an unread count helper for feed source list rendering.
- Modify `packages/db/src/repositories/feed-entries.ts`: upsert helper and mark unread helper.
- Modify `apps/worker/src/index.ts`: start worker.
- Create `apps/worker/src/workers/feed-worker.ts`: process feed fetch jobs.
- Create `apps/worker/src/lib/feed-parser.ts`: parse RSS/Atom into normalized entries.
- Modify `packages/queue/src/queues.ts`: include `userId` in `FeedFetchJobPayload` if worker needs ownership for writes.

## Task 2: Feed Entry APIs

**Files:**
- Create: `apps/web/src/app/api/feeds/[id]/entries/route.ts`
- Create: `apps/web/src/app/api/feed-entries/[id]/route.ts`

- [ ] **Step 1: List entries for a feed**

Before returning entries, verify the feed belongs to the current user. Then return entries ordered by `publishedAt desc, createdAt desc`.

- [ ] **Step 2: Read and mark entry state**

`GET /api/feed-entries/[id]` returns a user-owned entry. `PATCH` accepts `{ read: boolean }`; `read: true` sets `readAt = new Date()`, `read: false` sets `readAt = null`, both increment `version`.

- [ ] **Step 3: Add route smoke test**

Add `tests/unit/feeds-api.test.mjs` following `tests/unit/notes-api.test.mjs`. Document the required dev server and test user in the file header.

- Modify: `apps/worker/src/index.ts`
- Modify: `packages/db/src/repositories/feed-entries.ts`

- [ ] **Step 1: Add idempotent upsert helper**

In `feed-entries.ts`, add an `upsertByFeedUrl(userId, input)` helper using Prisma `upsert` or `findFirst` then create/update. It must not create duplicates for the same `feedId + url`.
