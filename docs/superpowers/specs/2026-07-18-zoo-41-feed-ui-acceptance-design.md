# ZOO-41 订阅验收 UI 优化设计

## 背景

ZOO-35 已把订阅刷新改成数据库排队与服务器 Cron，但订阅界面仍保留旧同步刷新文案；同时新增订阅弹窗、列表栏目操作和阅读元信息与当前产品层级不一致。历史 ZOO-20 要求在栏目标题区增加三个点，本次用户明确推翻该决定：栏目头部只负责导航、搜索和新增，文章操作必须归属于具体条目。

## 目标行为

左侧订阅源菜单收到 `202` 且响应为 `{ queued: true }` 时只提示“已安排更新，后台将在一分钟内处理”，不能在 Cron 执行前宣称没有新文章。新增订阅弹窗在没有选中结果时继续使用原生 `disabled`，并通过语义色降低背景、前景和光标强调；选中结果后恢复主按钮外观。

订阅列表栏目头部删除文章操作三个点。每篇订阅文章使用与剪藏卡片一致的外层容器和右下角 `CardActionMenu`；菜单提供收藏和复制链接。已收藏书签在非悬停状态继续作为状态标识，悬停或菜单打开时隐藏，为操作按钮让位。阅读器工具栏仍保留文章操作。

订阅正文元信息只保留作者、来源、发布时间和原文链接，删除字数与预计阅读时间。分隔符继续由渲染顺序生成，避免首尾或连续圆点。

## 边界

本 Issue 不修改 Cron、Feed API、数据库 Schema、AI Summary、Clip 卡片行为或全局按钮设计。只在订阅页面、共享卡片菜单和现有样式系统内完成变化。

## 验收

- 空弹窗和零选择状态下，添加按钮不可点击且视觉降级；选择结果后恢复。
- 左侧源刷新显示排队文案，失败仍显示错误。
- 订阅栏目头部没有三个点，文章卡片右下角有三个点，剪藏卡片不回归。
- 收藏标记与文章操作按钮不重叠。
- 正文元信息不含字数和预计时间，作者、来源、时间、原文仍存在。
- 深色、浅色与窄窗口通过浏览器检查。
