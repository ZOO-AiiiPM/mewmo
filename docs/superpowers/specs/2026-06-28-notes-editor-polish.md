# Spec: Notes 编辑器优化（Crepe 打磨第一批）

> 范式已定：Notion 式 WYSIWYG（Crepe）。本批为体验打磨，不改范式。

## 背景

Crepe 基线已上（commit 1333fa6）。实测发现几处体验问题，按协作流程做第一批优化。
能力边界已读 Crepe 7.21.2 源码证实，方案贴合库的原生机制，不在其上造封装。

## 要做什么（4 项，按确定性排序）

### 1. Heading 字体：衬线 → 黑体
- **现状**：frame 主题 `--crepe-font-title: 'Noto Serif'`，标题显示为衬线体，与产品调性（干净黑体）不符。
- **目标**：标题用项目 sans 字体（`--font-sans`），正文不变。
- **为什么**：用户明确要 Image #9 那种黑体标题，衬线「太奇怪」。

### 2. 拖拽柄（block handle）hover 可靠出现
- **现状**：柄（`+` 新增 / `⠿` 拖拽）是 Crepe 单个浮动元素，hover 跟随。但编辑器左 padding 仅 24px + 外层 `overflow:auto`，柄渲染在左侧 gutter 被裁切，导致「以为没有此功能」。
- **目标**：保持 Crepe 原生 hover 范式（不做每块常驻），但留足左 gutter 让柄稳定可见，鼠标移到任意块都能稳定唤出。
- **为什么**：用户确认要 hover 式（Notion 原生），只是要它「正常显示」。

### 3. 底部胶囊工具栏（quick insert）
- **现状**：插入块只能走 `/` 斜杠菜单，发现成本高。Crepe 的 Toolbar 是选区行内格式条，无块级插入。
- **目标**：编辑器底部居中、胶囊状、半透明（backdrop-blur）的浮动工具栏，按钮一键插入：待办 / 引用 / 表格 / 代码。复用 Crepe 斜杠菜单同款命令（`commandsCtx` + `wrapInBlockTypeCommand` / `addBlockTypeCommand`）。
- **为什么**：用户指定此设计，作为 `/` 之外的显式入口。

### 4. 键盘快捷键
- **目标**：常用块的快捷键，与底部工具栏共用同一套插入逻辑（DRY）。
  - ⌘⇧7 待办、⌘⇧9 引用、⌘⇧T 表格、⌘⌥C 代码块。
- **为什么**：用户要键盘流，手不离键盘。

## 不做什么
- **高亮（==高亮==）**：Crepe 原生不支持，需自定义 mark 插件，用户确认本批不做。
- **每块常驻拖拽柄**：用户选了 hover 式，不做常驻。
- **暗色配色精映射**：留后续批次（本批先把上述 4 项做对）。

## 验收标准（DoD）
1. 标题为黑体（非衬线），正文不变。
2. 鼠标移到任意一个 block（标题/正文/列表/引用），左侧都能稳定出现 `+`/`⠿` 柄。
3. 底部胶囊工具栏可见、半透明居中；点待办/引用/表格/代码各自正确插入对应块。
4. 四个快捷键各自插入对应块。
5. 自动保存、刷新往返不受影响；无 console 报错。
