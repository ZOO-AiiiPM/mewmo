# 025 - 按产品形态做 me-too 调研会漏掉真实竞争对手

## 症状

围绕 mewmo 做 me-too 自检、竞品调研或市场判断时，如果只按 reader / 笔记 / 本地 AI 这类产品形态切片，第一轮结论可能显得完整且乐观，但会漏掉用不同叙事解决同一用户痛点的产品。

## 根因

mewmo 的用户旅程不是单一产品形态，而是捕获、整理、激活、消费、沉淀五个环节。腾讯 ima、NotebookLM、Apple Intelligence、Khoj 等产品可能不用“AI 信息管家”的包装，却覆盖同一组需求。niche 视角的调研还会天然过滤大厂免费产品、系统级集成和即将发布的能力。

## 修法

调研必须按捕获 / 整理 / 激活 / 消费 / 沉淀五个用户痛点维度独立展开，并在 prompt 中显式列出 Apple、Google、腾讯、字节、微软、OpenAI、Anthropic 等大厂的免费产品、即将发布产品和系统级集成。第一轮乐观结论默认不可信，要用不同切入维度做第二轮交叉验证，并主动探索“非典型对手”。

## 踩坑记录 / 可复用教训

乐观结论常常只是“没看到威胁”，不等于威胁不存在。悲观结论如果指向具体威胁，通常比乐观结论更值得先相信。me-too 自检结论应冷藏 24 小时，给用户反向 challenge 调研维度完整性的窗口。
