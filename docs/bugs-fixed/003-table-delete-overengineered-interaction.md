# 表格删除交互过度复杂化

- **症状**：表格删除交互越做越复杂
- **根因**：微功能被否后继续引入更复杂交互和资源，而不是回到最小实现
- **修法**：最终用现有 +/− 风格的 `×` 按钮删除整表，并清理复杂方案
- **关联文件**：`app/src/lib/livePreview.ts`, `app/src/index.css`
- **日期**：2026-05-29

## 踩坑记录

- 第一次尝试：加了右键菜单 + 多级确认 → 被否，交互太重
- 第二次尝试：加了拖拽手柄 + hover 动画 → 更复杂，依然被否
- 最终方案：用项目已有的 +/− 按钮风格加一个 × 按钮，最小实现
- **教训**：被否时往简单退，不要往复杂加。见 `lessons/ui-微功能-被否往简单退.md`
