1 · 产品定位 & 北极星

北极星句子 本地 AI 第二外脑——它知道你的私密 context（笔记、剪藏、待办、日历、健康），而云端工具永远做不到。

市场上最强的竞品（腾讯 ima、NotebookLM、Readwise）都是云端工具。这意味着用户永远不会把 感情日记、待办清单、健康数据、私密灵感放进去。 本地优先突破了这道心理防线，也是差异化护城河。

我们做什么

  • 本地笔记 + 剪藏一体化
  • AI 自动织网（捕获→关联旧内容）
  • 纯本地 SQLite，不上云
  • macOS 原生体验（vibrancy / 红绿灯）

我们不做什么（v1）

  • 登录 / 账号体系
  • 多端同步 / 云存储
  • 任务管理 / 日历（v3+）
  • RSS 订阅（v2）
差异化核心（必须守住） "AI 织网"：剪一篇文章进来，5 秒内 AI 自动找出你过去笔记里最相关的 3 条， 一键建双向链接。这是 ima / Readwise 做不到的（没有你的私密笔记语料）。

4.2 AI 织网（v1 核心差异化）详解

30 秒 Demo Loop（用户验收标准）

  1. 粘贴一篇公众号 / 网页 URL 进剪藏
  2. 5 秒内 AI 分析内容，右侧 AIPanel 弹出 Top 3 相关旧笔记
  3. 用户点击"建链"，自动在两侧笔记底部插入 Markdown 双链
  4. 30 秒内完成全程，无需配置
技术路径 本地向量嵌入（Ollama 本地模型 OR BMC API）→ SQLite 存向量 → 余弦相似度 Top-K → Rust 后端计算（不过前端 / 不走网络）

4.3 数据模型（SQLite schema 核心表）

表名核心字段说明
notesid, title, content_md, created_at, updated_at笔记 / 剪藏主表
tagsid, note_id, nameAI 自动标签
linksid, source_id, target_id, created_at双向链接关系
embeddingsid, note_id, vector BLOB本地语义向量，AI 织网用
eventsid, action, note_id, ts本地埋点，不上报

8 · 成功指标

30 秒捕获

新用户首次打开 → 完成第一次有价值的捕获动作（写笔记 / 剪藏 URL）≤ 30 秒。 这是 v1 验收的 P0 门槛。

Loop 完整率

周活用户中，完整跑完"捕获→整理→消费"至少一次的比例 ≥ 60%。 数据来源：本地 events 表。

AI 织网触发率

剪藏/新建后 AI 弹出相关笔记，用户点击"建链"的转化率 ≥ 30%。 上线 7 天内必须回看决定保留/调整。
