# "听不懂人话" 的根因 + 矫正

整个 session 我让用户重复说同一件事 5-7 次，反复推翻自己的方案。本文不是道歉，是把模式拆开来记，下次别再犯。

---

## 案例回顾

### 案例 1：sidebar 折叠态对齐（7 轮反转）

**用户诉求时间线**：
1. "收缩前对称，收缩后不对称"
2. "光亮区偏导致 icon 偏。光亮区与左边没有边距"
3. "收缩后水平位置不要变化"
4. "没有变化，收缩后变窄了！！！！"
5. （Playwright 测量）"这在干嘛"
6. "啥意思。现在收缩后全部都变拥挤了，水平位置都发生了变化，这和中间也没任何关系！！！！！！！"

**我做了什么**：每轮都改单点几何参数（w-8、pl-2、items-center、w-10 inline、40×40 cell、grid place-items-center…），每次"看起来更居中了"，每次又被推翻。

**真正诉求**：折叠态和展开态的容器结构应该完全同构（同 padding、同 height、同 margin），只在按钮内部条件渲染 label/count。我每次把折叠态当"特殊状态"做不同尺寸（h-8 vs h-10、mb-1 vs mb-2、pl-2 vs px-3），结果整体布局漂了。

### 案例 2：日期分组 header 局部诉求被全局改

**用户诉求**："顶部不要加粗" / "顶部不要加线条"

**我做了什么**：把整组 header 的 `font-semibold` / `border` 全删了。

**真正诉求**：是**第一个 header**（紧贴窗口顶部那个）的视觉处理特殊化，用 `idx === 0` 区分，不是把规则全局改掉。

### 案例 3：当前 message 图片看不到却硬猜

**用户诉求**："将这个元素与日期对齐..."（带图）

**我做了什么**：图被 summary 吞了，我编了 5 种候选解释（搜索元素？日期 header？空状态？新建按钮？），自信选了一种实施。

**应该做什么**：直说"图我看不到，能描述一下'这个元素'指的是哪个"。承认看不到比硬猜五个候选高效。

## 根因模式（按危害排序）

### 模式 A：诉求层级误判（最常见、最致命）

用户说 "X 不对" 时有 3 种层级，我默认按最低层级解释：

- **L1 参数**：某个值错了（"icon 应该 x=16 不是 x=12"）
- **L2 位置**：某些位置特殊化（"第一个不要线"，不是"全部都不要线"）
- **L3 结构规则**：状态间规则一致性（"折叠态和展开态应该同构"，不是"折叠态再调一个参数"）

L3 诉求被按 L1 处理 → 反复调参数都对不上 → 反复反转。

**触发信号**：用户说"位置变了 / 不一致 / 不对称 / 收缩后 X" → **默认是 L3**，先想"两个状态之间什么规则裂了"，不是"哪个参数错了"。

### 模式 B：第 2 次没改对就该停下来

错一次 = 可能是细节疏忽。**错两次 = 方法系统漏**，再试只会继续漏。

我每次错都"再试一个参数"，把次数当解法。第 2 次起应该停下来问："我理解你的意思是 X，对吗？" 而不是闷头继续。

确认成本 (10s) << 错档实施 + 推翻重写成本 (5min × N 轮)。

### 模式 C：局部反馈被全局化

用户指着一个元素说"X 不对"，默认是**局部修**，不是**全局规则重写**。

要全局改之前必须先确认："这是只针对当前位置，还是所有同类元素？"

### 模式 D：不可见信息硬猜

图片缺失 / 上下文截断 / 引用看不到 → **直说看不到**，不硬猜。猜得自信只会让用户花精力把我拉回来。

### 模式 E：重型工具滥用作"表演专业"

启动 Playwright 测量像素？用户没要。**理解错诉求时，工具不解决问题，只是把简单问题复杂化**。用户说"看不懂"才需要工具。

## 矫正动作清单

> 触发某个信号时强制执行对应动作。不是"记住"，是"看到信号就停"。

| 信号 | 动作 |
|------|------|
| 用户说"位置变了 / 不一致 / 不对称" | 默认按 L3 结构规则解释，不调参数。先问"两个状态之间什么规则裂了" |
| 第 2 次方案被推翻 | **停下**，反述理解（"我理解的是 X，对吗"），等用户校准再动手 |
| 用户指着具体位置说"X 不对" | 默认局部修。要全局改前先确认范围 |
| 图 / 引用 / 上下文看不到 | 直说"看不到"。不硬猜 |
| 想启动重型工具（Playwright / 复杂测量） | 先问"用户要的是这个吗"，不预防性启动 |
| 同一争议进入第 3 轮 | 完全停下，让用户描述"正解长什么样"，不再提方案 |

## 一句话记忆

> 用户的"不对"通常是规则问题，不是参数问题。错两次就该停下问，别再调第三个参数。
