# 产品 me-too 自检：调研维度偏窄导致结论 180° 反转

## 起因

2026-05-20，"AI 信息管家" macOS 桌面 demo 即将进入 v1.1 实施前，用户主动叫停："我担心做的是市面已有的成熟方案，先别 implement，派个 Agent 去做 me-too 自检调研。"

于是开了一个后台 Agent，目标很明确：判断这个 demo 在 2026 年的市场上还站不站得住、要不要继续做。

## 经过：1 小时内的反转

### 第一轮：乐观结论

第一轮调研覆盖了三个看起来"最像本项目"的赛道：

- **剪藏 / 阅读后**：Cubox、Readwise、Pocket、Omnivore、Karakeep、Slax
- **笔记 + AI hybrid**：Reor、Obsidian + Smart Connections、Heptabase
- **本地优先派**：同上若干

返回的报告很有结论感："**不是 me-too，建议继续做**"，给出的差异化点是「本地 + Mac 原生 + 剪藏+笔记+AI 一体化 + AI 织网」。看起来很合理，几乎要按这个结论开干 v1.1 了。

### 用户提醒：维度不够

正要采纳结论时用户拍了一下桌子：

> "你这个调研维度太窄了。AI 视频总结呢？主题沉淀（'发芽'）呢？跨源 AI 助手呢？完整 Loop 呢？中文区呢？"

### 第二轮：结论被完全推翻

补齐这五个维度，派第二个 Agent 重做。返回的报告把第一轮的乐观判断打得粉碎：

- **腾讯 ima.copilot**——国内最强隐性对手，已经覆盖了本项目 70% 的功能面：多源捕获（含视频）+ 跨源问答 + 四端免费 + 绑微信生态。第一轮**根本没列入**。
- **NotebookLM** + Audio Overview：让"跨源 RAG + 主题沉淀"这条差异化直接失效。
- **AnythingLLM**：本地一键 RAG 已经做到位——"本地 + Mac 原生"在 2026 已经是**基础卫生**而不是差异化。
- **Apple Intelligence**（macOS 26）：即将白送给 1 亿 Mac 用户，操作系统级降维。
- **Mem.ai**：已经验证过"自动主题聚类"——结果是用户觉得鸡刂。这条产品假设本身就被市场打过脸。
- **Khoj**（34.6k ⭐ on GitHub）：Daily Brief 早就做了。

第一轮和第二轮**间隔不到 1 小时**，结论 180° 反转。如果用户没主动 challenge，这个项目大概率就在错误的市场判断上 implement 下去了。

## 教训：为什么会偏窄

### 1. 按"最像本项目的赛道"找对手 = 幸存者偏差

第一轮的隐含逻辑是"找 reader / 笔记 / 本地 AI 这种长得像我们的产品"。这种调研方式天然会**漏掉用不同叙事做同一件事的对手**——比如腾讯 ima 用"知识库"包装，但本质做的是 AI 信息管家 80% 的事。叙事不同，关键词搜不到，于是被系统性忽略。

### 2. 没有显式列"国民级 / 大厂免费产品"维度

Apple、Google、腾讯、字节、微软、OpenAI、Anthropic 这些大厂的 AI 笔记 / 知识库 / 总结类产品，因为"不像 niche 独立产品"，会被独立产品视角的调研自动过滤掉。但**它们恰恰是终结一个赛道差异化的杀手**——Apple Intelligence 一旦 ship，所有"本地 AI 笔记"赛道的差异化叙事都得重新讲。

### 3. 按"产品形态"调研而非"用户痛点维度"调研

第一轮问的是"市面上有什么 reader / 笔记 App"——这是产品形态视角，会被竞品的产品形态绑架。
正确的问法是"用户的**捕获 / 整理 / 激活 / 消费 / 沉淀**每一环都有什么解法"——痛点维度视角，每一环独立看竞争格局，才不会漏掉那些"长得不像但解的是同一个痛"的对手。

### 4. 单轮调研结论天然过度乐观

任何单一调研都有视野盲区，**单轮的乐观结论默认不可信**。需要用"不同维度并行 + 用户主动 challenge 维度完整性"做交叉验证，而不是看到一份完整的报告就当真。第一轮报告的"完整感"恰恰是误导——它在它划定的小圈子里很完整，但圈子本身画错了。

## 如何应用

下次做 me-too 自检（任何项目）按这套来：

1. **按用户痛点 5 维度独立展开**：捕获 / 整理 / 激活 / 消费 / 沉淀（或同等的核心动作分解），**每维度独立派 Agent 调研，不能合并**——合并就会回到"按产品形态找"的老路。
2. **每次调研 prompt 必须显式写**："请同时列出 Apple / Google / 腾讯 / 字节 / 微软 / OpenAI / Anthropic 等大厂在此领域的**免费产品或即将发布的产品**"。
3. **第一轮结论先冷藏 24 小时再行动**——给反思时间和用户反向 challenge 维度完整性的机会。
4. **看似乐观的"不是 me-too"结论必须用第二轮（不同维度切入）验证后再相信**。乐观判断默认不可信，悲观判断默认可信——这是反直觉但必要的偏置。
5. **显式列"非典型对手"**：用不同叙事做同一件事的产品（如"知识库"包装的 AI 信息管家、"会议记录"包装的多源捕获工具、"收藏夹"包装的 AI Reader）必须主动挖一遍。
